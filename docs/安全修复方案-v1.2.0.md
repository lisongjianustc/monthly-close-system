## 二、P2 修复 — 批量插入性能

### 2.1 问题根因

`flushBatch` 方法虽有 `BATCH_SIZE=1000` 控制攒批阈值，但内部实现为：

```java
// 当前实现 — 每条记录一次网络往返
private void flushBatch(List<ImportRecord> records) {
    for (ImportRecord record : records) {        try {
            recordMapper.insert(record);  // N次网络往返
        } catch (Exception e) {
            record.setStatus("INVALID");
            record.setErrorMsg(e.getMessage());
            recordMapper.insert(record);  // 失败记录也是N次网络往返
        }
    }
}
```

万条记录产生约 **2万次** 数据库网络往返，极大拖慢导入速度。

### 2.2 修复方案

**核心思路**: 将成功记录和失败记录分开处理，均使用批量操作

#### Step 1: 引入批量插入辅助方法

在 `ImportRecordMapper.java` 中添加批量插入方法：

```java
package com.mc.data.mapper;

import com.baomidou.mybatisplus.core.mapper.BaseMapper;
import com.mc.data.entity.ImportRecord;
import org.apache.ibatis.annotations.Insert;
import org.apache.ibatis.annotations.Param;
import org.apache.ibatis.annotations.Select;
import org.apache.ibatis.annotations.Update;

import java.util.List;

public interface ImportRecordMapper extends BaseMapper<ImportRecord> {

    // 批量插入（MySQL/MariaDB）
    @Insert("<script>" +
            "INSERT INTO import_record (batch_id, row_no, account_code, account_name, amount, " +
            "dimension1, dimension2, dimension3, status, error_msg) VALUES " +
            "<foreach collection='records' item='r' separator=','>" +
            "(#{r.batchId}, #{r.rowNo}, #{r.accountCode}, #{r.accountName}, #{r.amount}, " +
            "#{r.dimension1}, #{r.dimension2}, #{r.dimension3}, #{r.status}, #{r.errorMsg})" +
            "</foreach>" +
            "</script>")
    void insertBatchForMySQL(@Param("records") List<ImportRecord> records);

    // 批量更新状态
    @Update("<script>" +
            "UPDATE import_record SET status = CASE id " +
            "<foreach collection='records' item='r'>" +
            "WHEN #{r.id} THEN #{r.status} " +
            "</foreach>" +
            "END, error_msg = CASE id " +
            "<foreach collection='records' item='r'>" +
            "WHEN #{r.id} THEN #{r.errorMsg} " +
            "</foreach>" +
            "END, validate_time = NOW() " +
            "WHERE id IN " +
            "<foreach collection='records' item='r' open='(' separator=',' close=')'>" +
            "#{r.id}" +
            "</foreach>" +
            "</script>")
    void updateStatusBatch(@Param("records") List<ImportRecord> records);
}
```

#### Step 2: 重写 flushBatch

```java
private void flushBatch(List<ImportRecord> records) {
    if (records == null || records.isEmpty()) {
        return;
    }

    List<ImportRecord> successRecords = new ArrayList<>();
    List<ImportRecord> failRecords = new ArrayList<>();

    for (ImportRecord record : records) {
        successRecords.add(record);  // 先假设成功
    }

    try {
        // 批量插入成功记录
        recordMapper.insertBatchForMySQL(successRecords);
    } catch (Exception e) {
        // 批量插入失败，降级为逐条插入并记录失败
        for (ImportRecord record : records) {
            try {
                recordMapper.insert(record);
            } catch (Exception inner) {
                record.setStatus("INVALID");
                record.setErrorMsg("数据库写入失败: " + inner.getMessage());
                record.setId(null);  // 强制insert
                try {
                    recordMapper.insert(record);
                } catch (Exception ignored) {
                    // 二次失败无法挽救，跳过
                }
            }
        }
    }
}
```

#### Step 3: 重写 validateData 使用批量更新

```java
@Override
@Transactional
public void validateData(Long batchId) {
    List<ImportRecord> records = recordMapper.selectByBatchId(batchId);
    if (records == null || records.isEmpty()) {
        return;
    }

    List<ImportRecord> toUpdate = new ArrayList<>();
    for (ImportRecord record : records) {
        boolean invalid = false;
        String errorMsg = null;

        if (record.getAccountCode() == null || record.getAccountCode().isEmpty()) {
            invalid = true;
            errorMsg = "科目编码不能为空";
        } else if (record.getAmount() == null) {
            invalid = true;
            errorMsg = "金额不能为空";
        }

        if (invalid) {
            record.setStatus("INVALID");
            record.setErrorMsg(errorMsg);
        } else {
            record.setStatus("VALID");
        }
        record.setValidateTime(LocalDateTime.now());
        toUpdate.add(record);
    }

    // 批量更新，而非逐条 updateById
    if (!toUpdate.isEmpty()) {
        recordMapper.updateStatusBatch(toUpdate);
    }
}
```

### 3.2 JWT 密钥外置

## 四、修复优先级与排期

| 优先级 | 问题 | 工作量 | 风险 |
|--------|------|--------|------|
| P2-1 | flushBatch改为批量insert | 中 | 中（需验证批量SQL兼容性） |
| P2-2 | validateData改为批量update | 低 | 低 |
| P3-1 | Aviator沙箱配置 | 低 | 低 |
| P3-2 | JWT密钥外置 | 低 | 中（需兼容现有token格式） |
| P3-3 | 魔法字符串枚举化 | 中 | 低（需全量回归测试） |
| P3-4 | ObjectMapper注入 | 低 | 低 |
| P3-5 | 审计日志持久化 | 高 | 中（需新增表和Service） |

**推荐顺序**: P3-1 → P3-4 → P3-2 → P2-2 → P2-1 → P3-3 → P3-5
