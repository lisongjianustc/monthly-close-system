## 四、问题分析

### 1. 后端500错误（非阻塞性）

大部分500错误实际是**数据库约束导致**，而非代码逻辑错误：
- 新增组织/角色/期间时，如果编码重复，返回500
- 前端表单实际上能正常提交，只是后端拒绝了重复数据

### 2. E2E测试中的非bug项

- **F11 异常处理无处理按钮**：因为没有PENDING状态的异常数据，属于正常业务逻辑
- **F12 统计卡片数量为1**：UI设计差异，不影响功能使用

### 3. Console 400/500错误

在E2E测试过程中，部分模块Console出现400/500错误，但：
- 这些错误在手动测试时**未复现**
- 可能是测试脚本快速操作导致的数据状态不一致
