# 月结报表管理系统 - 前端功能测试标准操作规程（SOP）

> 文档版本：v1.0
> 更新日期：2026-04-25
> 测试级别：UI功能测试 / 端到端（E2E）验收
> 测试方式：浏览器手动操作

---

## 1. 测试环境准备

### 1.1 环境信息

| 项目 | 值 |
|------|-----|
| 前端地址 | http://localhost:3000 |
| 后端API地址 | http://localhost:18080 |
| 测试账号 | admin / admin123（系统管理员） |
| 浏览器要求 | Chrome 120+ / Firefox 120+ / Safari 17+ |
| 屏幕分辨率 | 1920×1080 或以上 |

### 1.2 启动服务

```bash
# 启动后端（终端窗口1）
export JAVA_HOME=/opt/homebrew/opt/openjdk@17
cd /path/to/monthly-close-system
java -jar mc-api/target/mc-api-1.0.0-SNAPSHOT.jar --server.port=18080

# 启动前端（终端窗口2）
cd /path/to/monthly-close-system/mc-frontend
npm run dev
```

### 1.3 前置检查清单

- [ ] 打开 http://localhost:3000，自动跳转到登录页
- [ ] 登录页正常渲染（标题、表单、按钮可见）
- [ ] Chrome DevTools > Network 中无红色failed请求
- [ ] Console 中无 Error 级别报错（Warning 可忽略）

---

## 2. 测试流程总览

```
登录页 → 首页(Dashboard) → 系统管理 → 流程引擎 → 数据导入 → 规则配置 → 异常处理 → 通知管理
```

每个模块测试完成后，返回首页，再进入下一个模块。

## 12. 常见问题排查

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| 登录页一直显示"登录中..." | 后端API未启动或CORS问题 | 确认后端 18080 端口正常，JWT Filter CORS 头已配置 |
| 表格一直 loading | 后端接口报错或网络不通 | 打开 DevTools > Network 查看接口返回 |
| 新增后列表不刷新 | 前端状态未更新 | 刷新页面（F5） |
| 进度条不动 | 导入异步任务未完成 | 等待更长时间，或检查后端日志 |
| 点击无反应 | 按钮 loading 中禁用 | 等待 loading 结束 |
